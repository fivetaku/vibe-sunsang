# 바선생 (Vibe Sunsang)

바이브코더를 위한 AI 멘토 에이전트.

Claude Code와 나눈 **모든 대화를 자동으로 수집, 변환, 분석**하여 더 나은 AI 협업자로 성장할 수 있게 도와줍니다.

---

## 이런 분을 위한 도구입니다

- 개발 지식 없이 AI로 코딩하는 **바이브코더**
- "나는 뭘 잘못하고 있는 거지?" 궁금한 사람
- 매주 내가 한 작업을 돌아보며 **AI 활용 실력을 키우고 싶은** 사람
- AI한테 "고쳐줘"만 반복하다가 지친 사람

---

## 어떻게 작동하나요?

```
Claude Code 대화 기록 (JSONL)
        ↓ 자동 변환
읽기 좋은 Markdown 파일
        ↓ 분석
멘토링 / 성장 리포트 / 패턴 발견
```

Claude Code는 여러분이 나눈 모든 대화를 `~/.claude/projects/`에 JSONL 파일로 저장합니다.
바선생은 이 파일들을 **읽기 좋은 Markdown으로 변환**하고, 프로젝트별로 정리한 뒤,
여러분의 요청 패턴, 실수 습관, 성장 지표를 분석합니다.

---

## 설치 방법

### 방법 A: Claude Code에게 시키기 (추천)

Claude Code가 이미 설치되어 있다면, 아무 프로젝트에서 이렇게 말하세요:

> https://github.com/fivetaku/vibe-sunsang 이 저장소를 내 홈 디렉토리에 클론하고, 폴더로 이동해서 /onboard 실행해줘

Claude Code가 알아서 다운로드, 설정, 초기 변환까지 해줍니다.

### 방법 B: 직접 설치

```bash
git clone https://github.com/fivetaku/vibe-sunsang.git
cd vibe-sunsang
claude
/onboard
```

### 온보딩이 하는 일

`/onboard`을 실행하면 다음이 자동으로 진행됩니다:

1. Claude Code 대화 기록(`~/.claude/projects/`) 탐색
2. 발견된 프로젝트에 읽기 좋은 이름 지정 (예: `-Users-me-my-app` → `my-app`)
3. 모든 JSONL 대화를 Markdown으로 첫 변환
4. 변환 결과 요약 (프로젝트 수, 세션 수, 가장 활발한 프로젝트)
5. 바로 시작할지 선택 (멘토링 / 성장 리포트 / 나중에)

## 핵심 기능

### 1. `/retro` - 대화 변환 + 분석 가이드

JSONL 로그를 Markdown으로 변환하고, 분석 방법을 안내합니다.

```
/retro              ← 새 대화만 변환 (증분)
/retro force        ← 전체 재변환
/retro 프로젝트명    ← 특정 프로젝트만 변환
```

**변환 결과:**
- 프로젝트별 폴더로 정리된 Markdown 파일
- 각 파일에 메타데이터 포함 (날짜, 토큰 수, 사용 모델, 도구 목록)
- 전체 인덱스(`INDEX.md`)로 한눈에 파악

변환 후 6가지 분석 템플릿을 제안합니다:
- 프로젝트 패턴 분석
- 팀 모드 퍼포먼스 리뷰
- 성장 & 학습 트래커
- 버그/실수 패턴 발견
- 토큰/비용 효율 회고
- 디버깅 루프 개선

---

### 2. `/mentor` - AI 활용 능력 코칭

4가지 모드로 맞춤형 멘토링을 제공합니다.

| 모드 | 트리거 | 하는 일 |
|------|--------|---------|
| **A: 요청 품질 코칭** | "요청 코칭해줘" | 내 요청이 얼마나 명확했는지 A~D 등급으로 채점. 나쁜 요청 → 좋은 요청 변환 예시 제공 |
| **B: 안티패턴 진단** | "뭘 잘못하고 있는지" | 8가지 나쁜 습관(고쳐줘 증후군, 무비판적 수용 등) 체크. 구체적 사례와 해결 전략 |
| **C: 개념 학습** | "이게 뭐야" | 최근 대화에 나온 개발 개념을 비유와 예시로 쉽게 설명 |
| **D: 종합 코칭** (기본) | "멘토링해줘" | 전체 점검. 요청 품질 + 안티패턴 + 레벨 판정 + 행동 계획 |

**멘토링 결과물:**
- 현재 레벨 판정 (Level 1~5)
- A~D 등급의 요청 품질 채점
- 3단계 행동 계획: 지금 당장 / 이번 주 / 한 달 안에
- 원하면 `90-exports/mentor-YYYY-MM-DD.md`로 저장

**채점 기준:**

| 등급 | 기준 |
|------|------|
| **A** | 무엇/왜/어디/제약 모두 포함, 예시까지 제공 |
| **B** | 무엇/왜 포함, 일부 컨텍스트 제공 |
| **C** | 무엇만 있음, 컨텍스트 부족 |
| **D** | 모호하고 구체적이지 않음 |

### 3. `/growth` - 성장 리포트 자동 생성

대량의 세션 데이터를 전용 AI 에이전트(`growth-analyst`)가 분석하여 성장 리포트를 자동 생성합니다.

```
성장 리포트 만들어줘           ← 범위 선택 후 생성
이번 달 성장 리포트            ← 최근 1달
프로젝트명 성장 분석           ← 특정 프로젝트
```

**범위 선택:**

| 범위 | 설명 |
|------|------|
| 이번 주 | 빠른 주간 리뷰 |
| 최근 2주 | 일반적인 리뷰 주기 |
| 이번 달 | 월간 성장 확인 |
| 특정 프로젝트 | 프로젝트별 심층 분석 |
| 분기 / 전체 | 장기 트렌드 |

**리포트에 포함되는 분석:**

1. **기본 통계** - 총 세션 수, 메시지 수, 토큰 사용량, 프로젝트별 분포, 모델 분포
2. **요청 품질 분석** - 구체적/모호한 요청 비율, 이해 추구 질문 비율, 컨텍스트 포함 비율
3. **안티패턴 탐지** - 고쳐줘 증후군, 무비판적 수용, 컨텍스트 생략, 반복 에러 패턴
4. **성장 지표** - 요청 구체성 변화 추이, 에러 자가 진단 빈도, 새로 배운 개념, 도구 활용 다양성
5. **레벨 판정** - 5단계 중 현재 위치
6. **이전 리포트 비교** - 레벨 변화, 요청 품질 트렌드, 안티패턴 개선 여부

리포트는 `90-exports/growth-report-YYYY-MM-DD.md`에 자동 저장됩니다.

---

### 4. 자유 분석 - 대화로 직접 분석 요청

명령어 없이 자연어로 다양한 분석을 요청할 수 있습니다:

**프로젝트 리뷰:**
> "이번 달 my-app 세션들을 분석해줘. 주요 작업 유형, 반복 에러, 도구 활용 패턴을 정리해줘."

**버그/실수 패턴:**
> "모든 프로젝트에서 내가 겪은 실수 패턴을 찾아줘. 예방 체크리스트로 만들어줘."

**팀 모드 리뷰:**
> "팀 모드 세션을 분석해서 내 지시가 모호했던 구간, 중복 작업, 병목을 찾아줘."

**비용 분석:**
> "프로젝트별 토큰 사용량과 모델 분포를 정리해줘. 토큰을 줄일 방법도 알려줘."

**디버깅 루프 개선:**
> "디버깅 세션들에서 실패→가설→실험→결과 루프가 잘 돌았는지 평가해줘."

**특정 세션 심층 분석:**
> "[파일명] 세션을 자세히 분석해줘. 내가 어디서 더 잘할 수 있었는지."

## 레벨 시스템

바선생은 당신의 AI 활용 수준을 5단계로 진단합니다:

| Level | 이름 | 특징 | 판정 기준 |
|-------|------|------|-----------|
| 1 | **Observer** | "만들어줘"만 요청, 결과를 그대로 수용 | 모호한 요청 60%+, 질문 거의 없음 |
| 2 | **Questioner** | "왜?"라고 물어보기 시작 | "왜?" 질문 증가, 에러 읽기 시도 |
| 3 | **Collaborator** | 구체적으로 요청, 대안을 질문 | 구체적 요청 50%+, 제약 조건 명시 |
| 4 | **Orchestrator** | 작업을 나눠서 지시, 검증 주도 | 작업 분할, 검증 요청, 아키텍처 참여 |
| 5 | **Conductor** | 여러 AI를 조율, 기술적 의사결정 | 멀티 에이전트 조율, 전략적 도구 활용 |

## 추천 루틴

### 매주 금요일 (15분)

```
1. /retro              ← 이번 주 대화 변환
2. 멘토링해줘           ← 이번 주 리뷰 (종합 코칭)
3. 행동 계획 실천        ← 다음 주에 시도할 것 1가지
```

### 매월 1회 (30분)

```
1. /retro              ← 전체 변환
2. 성장 리포트 만들어줘  ← 월간 성장 리포트
3. 이전 리포트와 비교    ← 자동으로 트렌드 분석
```

## 폴더 구조

```
vibe-sunsang/
├── CLAUDE.md                      ← 에이전트 동작 규칙
├── .claude/
│   ├── commands/
│   │   ├── onboard.md             ← /onboard (초기 설정 + 첫 변환)
│   │   └── retro.md               ← /retro (변환 + 분석 가이드)
│   ├── skills/
│   │   ├── mentor/SKILL.md        ← 멘토링 코칭 (4가지 모드)
│   │   └── growth/SKILL.md        ← 성장 리포트 (에이전트 위임)
│   └── agents/
│       └── growth-analyst.md      ← 성장 분석 전용 AI 에이전트
├── 10-scripts/
│   ├── convert_sessions.py        ← JSONL→Markdown 변환 스크립트
│   └── project_names.json         ← 프로젝트 이름 매핑 (/onboard에서 생성)
├── 20-knowledge-base/             ← 성장 지식 베이스 (6개 문서)
├── 40-conversations/              ← 변환된 대화 (자동 생성, git 제외)
└── 90-exports/                    ← 리포트 저장 (자동 생성, git 제외)
```
