# 기획/아이디에이션 성장 추적 & 측정

> 성장은 느낌이 아니라 측정이다. 이 문서는 기획자로서 AI와 협업하는 능력이 실제로 발전하고 있는지를 추적하는 방법을 담는다.

---

## 레벨 시스템: 나는 지금 어디에 있는가

### L1 — Dreamer (꿈꾸는 사람)

**특징**:
- 요청이 막연하고 구체성이 없다
- "좋은 거 만들어줘", "트렌디하게 해줘" 수준
- AI 결과물을 그대로 최종본으로 사용
- 타깃이 "모든 사람"이거나 정의되지 않음
- 기획과 아이디어의 차이를 잘 모름

**AI 대화 예시**:
> "인스타 콘텐츠 기획해줘. 요즘 잘 되는 걸로."

**다음 레벨로 가려면**:
목적, 대상, 제약을 요청에 포함하는 습관 만들기.

---

### L2 — Shaper (형태를 만드는 사람)

**특징**:
- 목적과 대상을 명시할 수 있다
- 제약 조건을 일부 포함함 (예: "2주 안에", "예산 없이")
- AI 결과물에 간단한 피드백을 줄 수 있다
- 핵심 가치보다 기능 목록에 집중하는 경향

**AI 대화 예시**:
> "30대 직장인 여성을 위한 인스타 콘텐츠 기획해줘. 뷰티 브랜드인데 2주치 캘린더로."

**다음 레벨로 가려면**:
기획에 프레임워크를 도입하고, 대안을 비교하는 습관 만들기.

---

### L3 — Planner (계획하는 사람)

**특징**:
- 구조화된 기획을 작성할 수 있다 (문제 → 솔루션 → 실행)
- 페르소나를 활용하고, 사용자 관점에서 생각한다
- 프레임워크를 도구로 활용한다 (BMC, 여정 맵 등)
- 여러 번 피드백 루프를 돌린다
- 실행 계획이 포함된 기획을 낸다

**AI 대화 예시**:
> "출산 후 복직한 30대 초반 여성 페르소나로, 뷰티 브랜드의 2주치 인스타 콘텐츠 캘린더를 짜줘. 사용자 여정에서 인지 단계에 집중하고, 각 콘텐츠의 목적을 명시해줘."

**다음 레벨로 가려면**:
대안을 비교하고 실현 가능성을 검증하는 습관. 가설을 명시하는 연습.

---

### L4 — Strategist (전략가)

**특징**:
- 대안을 비교하고 선택 근거를 설명할 수 있다
- 기획에 검증 가능한 가설이 포함됨
- 실현 가능성을 제약 조건과 함께 검토한다
- 이해관계자 관점을 포함한다
- OKR/KPI 수준의 성과 지표를 정의한다

**AI 대화 예시**:
> "뷰티 브랜드 인스타 전략 3가지 방향을 비교해줘: 교육 콘텐츠형, 커뮤니티형, 인플루언서 협업형. 각각의 장단점, 필요 리소스, 3개월 내 달성 가능한 KPI 기준으로 비교하고 우리 상황(팀 2명, 월 예산 50만 원)에 맞는 추천을 해줘."

**다음 레벨로 가려면**:
시스템적 사고, 2차 효과 예측, 전략적 의사결정 프레임워크 도입.

---

### L5 — Visionary (비전을 가진 전략가)

**특징**:
- 전체 시스템을 보고 기획한다 (2차, 3차 효과 예측)
- 단기 실행과 장기 전략을 연결한다
- 기획이 비즈니스 목표에 정렬되어 있다
- AI를 사고 파트너로 활용한다 (검증, 비판, 탐색)
- 기획 과정 자체를 지속적으로 개선한다

**AI 대화 예시**:
> "뷰티 브랜드의 올해 SNS 전략을 OKR로 설계해줘. Q1은 인지도 확대, Q2는 커뮤니티 활성화, Q3는 전환율 개선, Q4는 리텐션 강화로 설정하고 싶어. 각 분기별 핵심 활동, KR, 그리고 분기 간 연결 로직을 잡아줘. 각 전략의 잠재 리스크도 포함해."

## 성장 지표 1: 기획 품질 지표

### 1-1. 구체성 점수

요청 또는 기획서를 작성한 뒤 아래 항목을 체크한다.

| 항목 | 미충족 (0) | 부분 충족 (1) | 완전 충족 (2) |
|------|-----------|--------------|--------------|
| 목적이 한 문장으로 정의되어 있다 | | | |
| 타깃 사용자/독자가 구체적으로 명시되어 있다 | | | |
| 주요 제약 조건이 1개 이상 포함되어 있다 | | | |
| 기대 결과/성공 기준이 명시되어 있다 | | | |
| 맥락(배경, 현재 상황)이 포함되어 있다 | | | |

**점수 해석**:
- 0~3점: L1~L2 수준. 요청의 구체성을 높이는 연습이 필요하다.
- 4~6점: L3 수준. 전반적으로 구체적이나 일부 요소가 빠진다.
- 7~10점: L4~L5 수준. 높은 구체성으로 AI와 협업하고 있다.

### 1-2. 대상 명확성 점수

기획서나 콘텐츠를 보고 답해보자:

- [ ] 타깃이 정의되어 있는가?
- [ ] 타깃의 주요 문제/욕구가 포함되어 있는가?
- [ ] 타깃이 이것을 왜 원하는지 설명되어 있는가?
- [ ] 타깃이 아닌 사람이 명시되어 있는가? (선택)

체크 수 × 25 = 대상 명확성 점수 (%)

### 1-3. 제약 조건 포함률

최근 기획 5건을 돌아보며 측정한다:
- 예산 제약 포함: ___ / 5
- 시간/일정 제약 포함: ___ / 5
- 인력/기술 제약 포함: ___ / 5

포함률 = (총 체크 수 / 15) × 100 = ___%

## 성장 지표 2: 구조화 지표

### 2-1. 프레임워크 활용률

최근 한 달 기획 작업에서:

| 프레임워크 | 활용 횟수 |
|-----------|---------|
| 페르소나 설정 | |
| 문제 → 솔루션 구조 | |
| 사용자 여정 맵 | |
| 비즈니스 모델 캔버스 | |
| 우선순위 매트릭스 | |
| OKR / KPI | |
| 기타 (직접 기입) | |

총 활용 횟수: ___ 회

### 2-2. 논리적 연결성 자가 평가

기획서 또는 AI에게 보낸 요청을 돌아보며:

> "이 기획의 [문제 정의] → [솔루션] → [실행 계획]이 자연스럽게 연결되는가?"

- [ ] 문제가 명확히 정의되어 있다
- [ ] 솔루션이 그 문제에 직접 대응한다
- [ ] 실행 계획이 솔루션을 구현한다
- [ ] 각 단계 사이에 논리적 비약이 없다

### 2-3. 우선순위 설정률

기획에 우선순위가 포함된 비율을 추적한다.

> "이 기획에서 가장 중요한 것 하나를 꼽으라면?"

이 질문에 바로 답할 수 있으면 우선순위가 설정된 것.

최근 기획 5건 중 우선순위가 명확했던 것: ___ / 5 = ___%

## 성장 지표 3: 실현성 지표

### 3-1. 실행 계획 포함률

기획서 5건 중 실행 계획(누가, 무엇을, 언제까지)이 포함된 것: ___ / 5

### 3-2. 검증 요청률

AI와 대화할 때, 결과물의 약점이나 가정을 검토하도록 요청한 횟수:

> "이 기획의 가장 큰 약점은?" / "이 가정이 틀렸다면?" / "대안은?"

최근 10회 AI 대화 중 검증 요청을 포함한 것: ___ / 10 = ___%

### 3-3. 반복 개선 횟수

하나의 기획/콘텐츠를 완성하는 데 평균 몇 번 수정했는가:
- 1회 (첫 결과 그대로 사용): L1 수준
- 2~3회: L2~L3 수준
- 4~6회: L4 수준
- 7회 이상 + 구조적 개선: L5 수준

최근 한 달 평균 반복 횟수: ___ 회

## 레벨업 기준표

| 현재 레벨 | 다음 레벨 조건 |
|---------|--------------|
| L1 → L2 | 3회 연속으로 목적 + 대상 + 제약을 포함한 요청 작성 |
| L2 → L3 | 프레임워크를 활용한 기획 3건 완성, 평균 3회 이상 반복 개선 |
| L3 → L4 | 대안 비교가 포함된 기획 3건, 가설 명시 + 검증 요청 포함 |
| L4 → L5 | OKR 기반 전략 기획 1건, 이해관계자 분석 포함, 시스템 효과 예측 포함 |

## 분기별 회고 질문

분기 1회, 아래 질문에 답하며 더 깊은 성장을 점검한다.

1. 이번 분기 가장 잘 된 기획은 무엇이고, 왜 잘 됐는가?
2. 실행까지 이어지지 않은 기획의 공통적인 이유는 무엇인가?
3. AI를 활용하는 방식이 분기 초와 비교해 어떻게 달라졌는가?
4. 어떤 안티패턴을 극복했고, 어떤 안티패턴이 아직 남아있는가?
5. 다음 분기에 새로 도입하고 싶은 프레임워크나 접근 방식은?

---

> 측정하지 않으면 성장을 느낌으로만 판단하게 된다. 느낌은 종종 틀린다. 기록하고 비교하면 방향이 보인다.
