# AI 코딩 도구에 효과적으로 요청하는 법

> 코딩 지식이 없어도 명확하게 요청하면 AI가 더 좋은 결과를 만듭니다

## 핵심 원칙

### 1. 구체적으로 말하기 (Be Specific)

| 나쁜 요청 | 좋은 요청 |
|-----------|-----------|
| "웹사이트 만들어줘" | "React로 할 일 목록(Todo) 앱을 만들어줘. 할 일 추가, 완료 체크, 삭제 기능이 필요해" |
| "이거 고쳐줘" | "로그인 버튼을 클릭하면 '404 not found' 에러가 나. 라우팅 설정을 확인해줘" |
| "더 좋게 만들어줘" | "이 페이지의 로딩 속도가 느려. 이미지 최적화와 불필요한 API 호출을 줄여줘" |

### 2. 맥락 제공하기 (Provide Context)

요청 전에 AI에게 알려줘야 할 것:

```
프로젝트: [어떤 프로젝트인지]
현재 상태: [지금 뭐가 되어있고 뭐가 안 되는지]
목표: [최종적으로 원하는 것]
제약: [사용해야 하는 기술, 지켜야 할 규칙]
```

### 3. 한 번에 한 가지 (One Thing at a Time)

큰 작업을 작은 단계로 나누세요:

```
Step 1: "먼저 데이터베이스 스키마를 설계해줘"
Step 2: "이제 API 엔드포인트를 만들어줘"
Step 3: "마지막으로 프론트엔드 UI를 만들어줘"
```

### 4. 예시 보여주기 (Show Examples)

```
"사용자 프로필 카드를 만들어줘.
참고로 이런 느낌이야:
- 왼쪽에 아바타 이미지
- 오른쪽에 이름, 이메일
- 아래에 '프로필 편집' 버튼
Twitter의 프로필 카드랑 비슷한 레이아웃"
```

## 상황별 요청 템플릿

### 새 기능 요청

```
[기능명]을 만들어줘.

목적: [왜 필요한지]
입력: [사용자가 뭘 하는지]
출력: [어떤 결과가 나와야 하는지]
제약: [기술 스택, 스타일 가이드 등]

참고: [비슷한 기능이나 예시]
```

### 버그 수정 요청

```
[어떤 기능]에서 문제가 있어.

현상: [어떤 문제가 발생하는지]
재현 방법: [어떻게 하면 문제가 나타나는지]
기대 동작: [원래 어떻게 동작해야 하는지]
에러 메시지: [있다면 복사해서 붙여넣기]
```

### 코드 이해 요청

```
[파일명/기능명]을 설명해줘.

특히 알고 싶은 것:
1. 이 코드의 전체 흐름
2. 왜 이런 구조로 되어있는지
3. 내가 수정할 때 조심해야 할 부분

비개발자도 이해할 수 있게 비유를 들어서 설명해줘.
```

### 리팩토링 요청

```
[파일명]을 개선해줘.

현재 문제:
- [구체적 문제 1]
- [구체적 문제 2]

지켜야 할 것:
- 기존 기능은 그대로 유지
- [특정 패턴/스타일]을 따라줘

변경 전에 계획을 먼저 설명해줘.
```

## 요청 품질 자가 진단

요청을 보내기 전에 이 체크리스트를 확인하세요:

### Must-Have (필수)
- [ ] **무엇을** 원하는지 명확한가?
- [ ] **왜** 필요한지 배경이 있는가?

### Should-Have (권장)
- [ ] **어디에** 적용하는지 구체적인가? (파일, 컴포넌트, 기능)
- [ ] **제약 조건**이 있으면 알려줬는가?
- [ ] **예시나 참고**를 제공했는가?

### Nice-to-Have (있으면 좋은)
- [ ] 성공 기준이 명확한가? ("이렇게 되면 완성")
- [ ] 우선순위가 있다면 표시했는가?

## 프로 팁

### AI에게 되물어보게 하기
```
"이 요청에 대해 네가 이해한 것을 정리해줘.
빠진 정보가 있으면 나한테 질문해줘.
작업은 아직 시작하지 마."
```

### AI의 결과를 검증하기
```
"방금 만든 코드를 네가 직접 리뷰해줘.
잠재적 문제점, 빠진 에러 처리, 보안 이슈가 있는지 확인해줘."
```

### 학습을 위한 요청
```
"이 변경사항을 적용하기 전에,
왜 이렇게 하는 건지 비개발자도 이해할 수 있게 설명해줘.
어떤 대안이 있었고 왜 이걸 선택했는지도 알려줘."
```
