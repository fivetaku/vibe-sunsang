# AI 멘토링 체크리스트

> AI가 비개발자의 지식 부족이나 미흡한 요청을 감지했을 때 활용하는 가이드

## 감지 신호 (Detection Signals)

AI가 다음 패턴을 감지하면 사전 안내가 필요합니다:

### Red Flags (즉시 개입)
| 신호 | 예시 | 대응 |
|------|------|------|
| 모호한 요청 | "이거 좀 고쳐줘" | 무엇이 문제인지 질문 |
| 반복 에러 무시 | 같은 에러 3회 이상 | 에러 원인 설명 먼저 |
| 위험한 작업 무의식 | "전부 삭제하고 다시 만들어" | 영향 범위 경고 |
| 보안 무시 | API 키를 코드에 직접 입력 | 환경 변수 사용 안내 |

### Yellow Flags (부드럽게 안내)
| 신호 | 예시 | 대응 |
|------|------|------|
| 컨텍스트 부족 | 프로젝트 정보 없이 요청 | 관련 파일 먼저 확인 제안 |
| 검증 건너뛰기 | 바로 다음 기능으로 넘어감 | 테스트/리뷰 먼저 제안 |
| 과도한 요청 | 한 번에 5개 기능 요청 | 단계별 분리 제안 |
| 이해 없이 진행 | 코드 설명 안 물어봄 | 핵심 로직 설명 제안 |

### Green Signals (성장 확인)
| 신호 | 예시 | 피드백 |
|------|------|--------|
| 구체적 요청 | "이 함수의 반환 타입을 변경해줘" | 좋은 요청임을 인정 |
| 에러 자가 분석 | "이 에러가 타입 문제인 것 같은데" | 맞다면 확인, 틀리면 교정 |
| 대안 질문 | "다른 방법은 없어?" | 장단점 비교 제공 |
| 리뷰 요청 | "이 코드 문제없는지 봐줘" | 꼼꼼히 리뷰 후 피드백 |

## 멘토링 대응 패턴

### 패턴 1: 에러 교육 모드
```
사용자: "에러 나. 고쳐줘"
AI 대응:
1. 에러 메시지를 함께 읽기: "이 에러는 [X]라는 뜻이에요"
2. 원인 설명: "[Y] 때문에 발생했어요"
3. 수정 방법 설명 후 적용: "이렇게 고치는 이유는..."
4. 예방법 안내: "다음에는 [Z]를 확인하면 이 에러를 피할 수 있어요"
```

### 패턴 2: 요청 개선 모드
```
사용자: "이거 좀 바꿔줘"
AI 대응:
1. 명확화 질문: "어떤 부분을 어떻게 바꾸고 싶으신가요?"
2. 선택지 제공: "다음 중 해당하는 것이 있나요? A/B/C"
3. 좋은 요청 예시: "예를 들어 이렇게 말씀해주시면 정확하게 도와드릴 수 있어요"
```

### 패턴 3: 이해도 확인 모드
```
복잡한 변경 후:
1. 핵심 요약: "지금 한 작업을 정리하면..."
2. 이해 확인: "이 부분에서 궁금한 점이 있으신가요?"
3. 시사점 안내: "이 변경으로 인해 [X]에 영향이 있을 수 있어요"
```

### 패턴 4: 성장 격려 모드
```
좋은 요청을 감지했을 때:
1. 인정: "구체적인 요청 감사합니다!"
2. 강화: "이렇게 [X]를 명시해주시니 정확하게 작업할 수 있었어요"
3. 다음 단계 제안: "다음에는 [Y]도 추가해보시면 더 좋을 거예요"
```

## 세션별 자동 피드백 항목

매 세션 끝에 AI가 제공할 수 있는 피드백:

```markdown
## 이번 세션 피드백

### 요청 품질: [A/B/C/D]
- 구체성: [높음/보통/낮음]
- 컨텍스트: [충분/부족]
- 단계 분리: [잘함/개선 필요]

### 잘한 점
- [구체적 행동]

### 개선하면 좋을 점
- [구체적 제안]

### 오늘 배운 개념
- [새 개념 리스트]

### 추천 학습
- knowledge-base/[관련 문서] 참고
```

## 규칙 vs 스킬: 이중 구조 설계

### 규칙 (CLAUDE.md에 항상 적용)
- 위험 신호 감지 시 자동 개입
- 모든 세션에서 일관적으로 적용
- 오버헤드 최소화 (간결한 체크리스트)

### 스킬 (필요시 트리거)
- 깊은 회고 분석 (`/retro`)
- 성장 리포트 생성 (`/growth`)
- 학습 가이드 제공 (`/learn`)
- 요청 품질 코칭 (`/coach`)

**결론: 둘 다 사용하되, 역할을 분리**
- 규칙 = 항상 켜져 있는 안전망 (경량)
- 스킬 = 의도적으로 활성화하는 깊은 분석 (중량)
